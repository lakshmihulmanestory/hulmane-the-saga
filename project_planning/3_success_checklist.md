# The "Success" Checklist

To keep a multi-year, weekend-only project alive and moving forward without burning out, adhere to these three foundational rules:

## 1. The World Saga
* **What it is**: A living document containing every rule, map, character trait, and family tree.
* **Why it matters**: Without this, the team (and you) will get lost in the complexity over time.
* **Execution**: Stored securely (in this repository or a wiki). Any new canon *must* be added here immediately.

## 2. Standardized Format
* **What it is**: All writing, lore entries, and notes must be submitted in a standardized format.
* **Why it matters**: It removes formatting friction. It ensures all content is easily convertible to websites, published books, scripts, or game wikis later.
* **Execution**: **Everything must be written in Markdown (`.md`)**. Use predefined templates for characters, locations, and storylines.

## 3. Micro-Milestones
* **What it is**: Setting extremely granular, easily achievable goals.
* **Why it matters**: Since development happens primarily on weekends, huge goals like "Finish Volume 1" are demoralizing and take too long to check off.
* **Execution**: Instead of "Finish Volume 1", set a goal like: *"Finish 2 Character Profiles this Sunday"* or *"Outline the prompt for Chapter 4."* Celebrate the small, consistent wins.
